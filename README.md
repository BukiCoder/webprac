Оранжевый цвет обозначает, что данные действия или страницы доступны только менеджеру.

## Описание страниц

### Шапка на каждой странице:
* Кнопка "войти"  ~~>  страница с формой авторизации
* Кнопка "зарегистрироваться" ~~> страница с формой регистрации
* Список вкладок (надписи, на которые можно нажать, клик на каждую переводит на соответствующую страницу): 
  * Главная страница
  * Список автомобилей
  * Список моделей
  * Список марок
  * Елси клиент авторизован, то на месте кнопок входа и регистрации располагается вкладка личного кабинета. 
  * МЕНЕДЖЕР: список заказов 
  * МЕНЕДЖЕР: список клиентов

### Главная страница
* Информация о салоне, фото салона
* Контактная информация

### Форма регистрациии клиента
* Поля ввода ФИО, email, адреса, номера телефона и пароля
* Проверочное поле ввода пароля
* Кнопка "зарегистрироваться" ~~> в случае успешной регистрации добавление пользователя в БД и переход на главную странцу. В случае ошибки - оповщение о причине.

### Форма авторизации
* Поля ввода ФИО, email/номера телефона и пароля
* Кнопка "войти" ~~> в случае успешной авторизации переход на главную странцу. В случае ошибки - оповщение о причине.

### Список автомобилей
* Меню поиска, в котором можно выбрать фильтры по характеристикам автомобиля, кнопка "применить фильтр".
* Карточки автомобилей:
  * Заголовок с названием модели и марки
  * Фото автомобиля
  * Цена
  * Краткие характеристики: мощность/обьем двигателя, цвет, год производства
  * МЕНЕДЖЕР: кнопка "удалить" ~~> удаляет автомобиль из БД
* МЕНЕДЖЕР: кнопка "добавить автомобиль" ~~> форма добавления автомобиля.
* На странице помещается до 10 карточек. Если не помещаютя, то появляются дополнительные элементы, которые позволют показать следующие/предыдущие 10 карточек.
* Клик на карточку переводит пользователя на страницу соответствующего автомобиля.

### Страница автомобиля
* Фото автомобиля, цена, характеристики автомобиля из БД: общие характеристики модели и комплектации + конретно данного автомобиля.
* Кнопка для оформления заказа, доступная только клиенту: переводит на форму оформления заказа.
* МЕНЕДЖЕР: кнопка "изменить" ~~> позволяет изменить любую характеристику автомобиля(не модели или комплектации), а также выбрать статус автомобиля.
* МЕНЕДЖЕР: кнопка "сохраннить изменения" ~~> обновляет БД.
* МЕНЕДЖЕР: кнопка "удалить" ~~> удаляет из БД и переход на страницу списка автомобилей.
* МЕНЕДЖЕР: Список клиентов с ФИО, id и датами проведения тест-драйвов.

### Форма оформления заказа
* Чекбокс нужен ли тест-драйв. Если нужен то активны поля дата и время тест-драйва, длительность
* Чекбокс нужна ли доставка, если нужна то активны поля адрес доставки и дата доставки
* Надпись с итоговой ценой
* МЕНЕДЖЕР: поле для указания id клиента
* Кнопка "оформииь заказ"

### Список моделей
* Меню поиска, в котором можно выбрать фильтры, кнопка "применить фильтр".
* Карточки моделей:
  * название модели и марка в зоголовке
  * фото + базовая стоимость
  * МЕНЕДЖЕР: кнопка "удалить" ~~> удаляет модель из БД
* На странице помещается до 10 карточек. Если не помещаютя, то появляются дополнительные элементы, которые позволют показать следующие/предыдущие 10 карточек.
* Клик на карточку ~~> страница с информацией о модели.
* МЕНЕДЖЕР: кнопка "добавить модель" ~~> форма добавления модели.

### Страница модели
* Фото модели, название, характеристики модели из БД.
* Список комплектаций, возможность просмотра информации о каждой их них (изначально вабрана первая из них и отображена информация о ней).
* Кнопка, которая переводит пользоватеоя на страницу с автомобилями с примененным фильтром по модели.
* МЕНЕДЖЕР: надпись с id модели.
* МЕНЕДЖЕР: кнопка "изменить" ~~> позволяет изменить каждую характеристику каждой комплектации и модели в целом.
* МЕНЕДЖЕР: кнопка "добавить комплектацию" ~~> добавляет новую комплектацию в список.

### Личный кабинет клиента
* Надписи с ФИО, email, номером телефона, адресом.
* Список заказов в виде карточек:
  * название автомобиля, VIN + фото
  * информация о доставке: тип(доставка/самовывоз). В случае доставки указан адрес доставки, дата
  * если заказ с тест-драйвом: дата и время тест-драйва, его длительность
  * статус заказа
  * кнопка "отменить заказ"

### Список марок
* Карточки с логотипом и названием. МЕНЕДЖЕР: кнопка удалить ~~> удаляет марку из БД.
* Клик на карточку ~~> страница марки.
* МЕНЕДЖЕР: кнопка "добавить марку" ~~> форма добавления марки.

### Страница марки
* Логотип, название, описание, год основания, контактная информация.
* Кнопка, которая переводит пользователя на страницу с моделями с примененным фильтром по марке.
* МЕНЕДЖЕР: кнопка "изменить" ~~> позволяет изменить данные 
* МЕНЕДЖЕР: кнопка "сохранить изменения" ~~> сохраняет данные в БД

МЕНЕДЖЕР:
### Форма добавления автомобиля
* Поля, позволяющие указать характеристики автомобиля, модель и комплектацию из списка
* Кнопка "добавить" ~~> добавляет автомобиль в БД, переводит на страницу автомобиля. 

### Форма добавления модели
* Аналогична странице модели в режиме редактирования + кнопка "добавить" ~~> добавляет модель в БД

### Список клиентов
* Меню поиска, в котором можно выбрать фильтры, кнопка "применить фильтр".
* Карточки клиентов:
  * заголовок с ФИО
  * id, email, номер телефона
  * кнопка "удалить" ~~> удаляет клиента из БД
  * клик на карточку ~~> страница клиента

### Страница клиента
* Аналогична личному кабинету, за исключением возможности редактирования(с помощью кнопки "изменить" + кнопка "сохранить") данных о клиенте и изменения статуса заказа на произовльный + кнопка "удалить" ~~> удаляет клиента из БД  + надпись с id пользователя. 

### Список заказов
* Меню поиска, в котором можно выбрать фильтры, кнопка "применить фильтр".
* Карточки заказов, аналогичные тем, что в личном кабинете, дополненные ФИО и id пользователя. Также есть возможность изменить статус закаказа на произвольный из списка. 

## Форма добавления марки
* Поля для ввода характеристик.
* Кнопка "добавить" ~~> добавляет марку в БД и переводит на страницу марки.

## Сценарии использования
### Клиент 
| Действие  | Последовательность действий |
|-------|---------------------|
| Регистрация | Пользователь переходит на главную страницу, нажимает на кнопку "зарегистрироваться", на форме регистрации вводит свои данные и нажимает кнопку подтверждения регистрации.|
| Авторизация | Пользователь переходит на главную страницу, нажимает на кнопку "войти", на форме авторизации вводит ФИО, email или телефон, пароль; нажимает кнопку подтверждения входа. |
| Получение списка автомобилей/поиск автомобиля | Клиент в шапке нажимает на надпись, которая переводит его на страницу со списком автомобилей, где он может настроить фильтры. Также клиент может перейти на страницу со списком моделей, где точно так же он может настроить фильтры и подобрать подходящую модель, после чего подобрать конкретный автомобиль данной модели.|
| Оформление заказа | Клиент нажимает на карточку понравивишегося автомобиля из списка автомобилей; если его всё устраивает, клиент нажимает на кнопку "заказать", далее на форме оформления заказа он указывает нужные ему опции и нажимает кнопку подтверждения. Далее заказу назначается менеджер, который будет отвечать за данный заказ, тест-драйв и общение с клиентом.|
| Проверка статуса заказа/отмена заказа | Клиент в шапке нажимает на вкладку личного кабинета, после чего в личном кабинете в списке заказов он может проверить статус заказов и отменить заказ, нажав кнопку "отменить" в карточке соответствующего заказа. |

### Менеджер
Менеджер имеет дополнительные возможности:
* Список марок автомобилей (добавить, изменить, удалить)
* Список моделей автомобилей (добавить, изменить, удалить)
* Список автомобилей (добавить, изменить, удалить)
* Список клиентов (добавить, изменить, удалить)
* Список заказов (изменить статус)

| Действие  | Последовательность действий |
|-------|---------------------|
| Получение списка клиентов по характеристикам их заказов | Менеджер нажимает в шапке на надпись списка заказов, настраивает фильтры, указывая необходимые параметры поиска, после чего получает подходящий список заказов, в которых указаны ФИО и id клиента. |
| Оформление заказа, проверка и изменение статуса заказа | Менеджер может добавить заказ аналогично тому, как это делает клиент, за исключением того, что в форме оформления заказа дополнительно указывается id клиента. Менеджер в шапке нажимает на надпись списка заказов, после чего для каждого заказа он може выбрать статус из списка. |
| Добавление и удаление клиента, чтение и редактирование данных о нем | Менеджер нажимает в шапке вкладку списка клиентов. На странице списка клиентов он может найти конкретного клиента, применив соответствующий фильтр. Удалить пользователя можно, нажава кнопку "удалить" на карточке клиента. Добавить пользователя можно нажава кнопку "добавить", после чего заполнить данные клиента в форме и нажать кнопку подтверждения. Редактировать данные можно нажав на карточку клиента, после чего нажать кнопку "изменить" на странице клиента, далее кнопку "сохранить изменения".|
| Добавление и удаление марки автомобилей или конкретного автомобиля, чтение и редактирование данных о них | Менеджер нажимает в шапке вкладку списка марок. На странице списка марок удалить марку можно, нажава кнопку "удалить" на карточке марки. Добавить марку можно, нажава кнопку "добавить", после чего заполнить данные марки в форме и нажать кнопку подтверждения. Редактировать данные можно нажав кнопку "изменить" на странице марке, после чего кнопку "сохранить изменения". Аналогично для автомобилей.|
